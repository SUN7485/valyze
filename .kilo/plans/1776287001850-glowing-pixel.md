# Plan: Cloud Report Management System

## Goal
Build a complete cloud-based report management system where:
1. Reports are saved to Supabase database
2. Users can view all saved reports from the system homepage
3. Users can load/edit/continue working on any saved report
4. Users can download reports in any format (PDF, JSON, XML, Excel, CSV, Word)

## Implementation Plan

### Phase 1: Backend - Export Service

#### 1.1 Create Export Service Module
- **File**: `backend/services/export_service.py`
- **Purpose**: Central service for generating all export formats
- **Functions**:
  - `export_json(report)` - Convert FullReport to JSON
  - `export_xml(report)` - Convert FullReport to XML
  - `export_excel(report)` - Generate XLSX using openpyxl
  - `export_csv(report)` - Generate CSV for tabular data
  - `export_word(report)` - Generate DOCX using python-docx

#### 1.2 Create Export API Endpoints
- **File**: `backend/api/export.py` (new)
- **Endpoints**:
  ```
  GET  /api/export/json/{report_id}
  GET  /api/export/xml/{report_id}
  GET  /api/export/excel/{report_id}
  GET  /api/export/csv/{report_id}
  GET  /api/export/word/{report_id}
  GET  /api/export/status/{report_id}
  ```

#### 1.3 Register Router
- **File**: `backend/main.py`
- Add: `from api.export import router as export_router`
- Add: `app.include_router(export_router)`

## Technical Details

### JSON Export
- Return raw report data as JSON
- Include fields + arrays + metadata

### XML Export
- Use Python's `xml.etree.ElementTree` or `lxml`
- Structure: `<Report><Fields>...</Fields><Arrays>...</Arrays></Report>`

### Excel Export
- Use `openpyxl` library
- Create sheets: Summary, Fields, Shareholders, Banking, etc.
- Style headers, enable filtering

### Word Export
- Use `python-docx` library
- Create document with tables for fields and arrays
- Professional formatting with styles

### CSV Export
- Use Python's built-in `csv` module
- Export main fields as flat CSV with key fields
