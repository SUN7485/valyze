# Automatic Empty Field/Section Hiding

## Goal
Automatically hide any field, section, or array that has no meaningful data across all outputs (PDF, preview, edit pages), while preserving edit-mode accessibility. Fix broken eye-icon visibility toggles and create a unified visibility system.

---

## Current Architecture Summary

**Data Flow:**
1. Frontend `FieldInput` components read/write via `ReportContext`
2. `updateField`/`updateArray` → backend CRUD → stores in `report_json`
3. PDF generation: `reportAPI.getReport()` → `PDFGenerator.get_html_preview()` → Mustache template → HTML → PDF

**Current Visibility Mechanisms:**
- **Section-level flags** (`show_egypt_fields`, `show_operations`, etc.) control entire blocks
- **Array loops** check `.length` (empty arrays skip rendering)
- **Manual hide** via eye icon stores field names in `report.arrays.hidden_fields`
- **Mustache conditionals** (`{{#field}}`) render only if value truthy
- **Backend NULL pruning** (commented out in pdf_generator)

**Critical Problems:**
1. Eye icon `hidden_fields` array stored but **never used by PDF generator**
2. Individual empty fields always render as "N/A" rows in registration grid (no per-field hiding)
3. No consistency: some sections empty but still render empty rows
4. No rule for what "empty" means – inconsistent across fields

---

## Root Cause Analysis

### Why Empty Fields Still Show
- Mustache template uses `{{key}}` which outputs "N/A" for missing fields
- Only section-level `{{#show_*}}` blocks prevent rendering; individual fields inside have no per-field conditionals
- No logic to suppress empty rows in grid layouts

### Why Eye Icons Do Nothing in Output
- `HiddenFields` array saved to DB → `report.arrays.hidden_fields`
- PDF generator never reads this array → eye state ignored
- No HTML token (`__HIDDEN__`) injection into template

---

## Implementation Approach

**Strategy: Centralized, Unified Visibility Layer**

1. **Define canonical "empty" check** – single function used everywhere
2. **Add per-field visibility flag** to context (`visible_fields` set or `hidden_fields` support)
3. **Update PDF generator** to respect hidden/empty visibility
4. **Make template conditionals field-aware** (wrap individual fields where needed)
5. **Keep eye icon behavior** but ensure it overrides auto-hide when user explicitly hides/shows
6. **Preserve edit-mode UX** – fields always visible in editor with prune button

**Key Design Decision:** Instead of field-level `{{#field}}` conditionals everywhere (which would be massive template changes), use **lxml HTML post-processing** to strip empty field containers based on unified visibility rules. This is cleaner, respects existing template structure, and applies universally.

- **File:** `backend/utils/visibility.py` (new)
- **Function:** `is_field_empty(value, field_name=None) -> bool`
- Normalizes: `None`, `""`, whitespace-only, `"N/A"`, `"NA"`, `"n/a"`, `"none"`, `"-"`, `"null"` (string)
- Returns `True` if empty, `False` otherwise
- Special handling: arrays/lists empty if `len == 0`; objects empty if all properties empty (defer to recursive check)
- Export for reuse in PDF generator and any other service

### 10. Update Preview Generation to Strip Empty Field DOM Nodes (backend/pdf_generator.py)

```python
from lxml import html, etree  # already imported?

def _strip_empty_fields(html_content: str, visible_fields: set[str]) -> str:
    """Remove field containers whose fieldName is not in visible_fields."""
    parser = html.HTMLParser()
    tree = html.fromstring(html_content, parser=parser)
    
    # Select all elements with data-field attribute
    for elem in tree.xpath('//*[@data-field]'):
        field_name = elem.get('data-field')
        if field_name not in visible_fields:
            # Find parent .reg-value or .metric-value or <td> and remove it + its label
            parent = elem.getparent()
            if parent is not None:
                # If parent is a value container, remove its sibling label too
                # Structure: <div class="reg-label">Label</div><div class="reg-value" data-field="...">...</div>
                # Both are siblings under same parent (usually .reg-grid)
                # Remove both elements
                label_sibling = parent.getprevious()
                if label_sibling is not None and 'reg-label' in (label_sibling.get('class') or ''):
                    parent.remove(label_sibling)
                parent.remove(parent)  # actually remove parent element
                # Alternatively, remove the parent container of both if wrapped
            else:
                # If element itself is the container, remove it
                elem.getparent().remove(elem)
    
    return html.tostring(tree, encoding='unicode', method='html')
```

Call it in `get_html_preview()` before returning.

**Verification:** Compare generated HTML before/after; missing fields removed; existing fields intact

### 12. Edge Cases & Special Handling

#### 12.1 Whitespace-only values
`is_field_empty()` strips whitespace before checking → treated as empty → auto-hide

#### 12.2 Mixed-case N/A values
`str(v).strip().upper() in ("N/A", "NA", "N/A", "NONE", "-")` → empty

#### 12.3 Required fields (e.g., company_name)
Required fields should **NOT** be auto-hidden in edit mode, but in PDF they should still hide if empty (report incomplete). Keep required validation on client; preview warns about missing required fields via existing completion%.

**No special handling needed** – they'll be hidden in final output if empty, which is correct (incomplete report). Editor shows required marker anyway.

#### 12.4 Rich text / HTML content
Some fields like `executive_summary_text` may contain HTML. Empty if contains only whitespace or `<p></p>`? Use a simple text extraction: `strip_tags(v)` then check empty. Add helper `is_rich_text_empty`.

#### 12.5 Optional images/files
Not present in current system. If added later, extend `is_field_empty` to check for empty file paths or missing uploads.

#### 12.6 "N/A" as actual value vs placeholder
Current system stores "N/A" as default for missing fields. Our rule treats "N/A" as empty → good. But if user intentionally enters "N/A", they may want it shown. We assume auto-hide is acceptable; eye icon can explicitly show it if needed.

#### 12.7 Sections that should always appear (headers, static text)
No data-field attribute → not stripped.

### 14. Backward Compatibility Risks

**Breaking changes:**
1. **Template HTML structure change** – added `data-field` attributes must not break existing CSS/JS. Non-breaking (new attributes ignored).
2. **Hidden field detection** – if user EXPLICITLY hid a field with value, it now hides (where before it showed). This is **desired behavior**; may surprise users who expected hidden fields to still show if they had data? Eye icon previously did nothing, so no existing habit.
3. **Empty field definition** – some fields may have used "N/A" as a sentinel value meaning "not requested" vs "not applicable". Since we auto-hide "N/A", they'll disappear. Verify all N/A uses are truly empty placeholders (seems so).
4. **Auto-hide may conflict with user expectation** of seeing all fields in preview even if blank. But user requested this feature.

**Mitigation:**
- Document that eye icon is now functional
- Provide clear UI feedback: Eye off = hidden from PDF; Prune = clears value
- Allow manual show via eye icon even if auto-hidden (user can re-show)
